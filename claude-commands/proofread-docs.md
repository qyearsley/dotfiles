# proofread-docs

Proofread and update documentation for consistency, clarity, and accuracy.

## Instructions

1. Find all documentation files (`*.md`) in the repository, excluding `node_modules`, `.venv`,
   `venv`, `dist`, `build`, and other common build/dependency directories.
2. Review each file for:
   - Consistency with codebase and other documentation
   - Clarity and readability
   - Accuracy of technical details
   - Proper markdown formatting
   - Outdated information that needs updating
   - Grammar and spelling
3. Make edits to improve the documentation where needed.
4. Summarize the changes made.
